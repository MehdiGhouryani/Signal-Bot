# فاز ۱ — بازطراحی ماژولار (Modular Restructure)

این فاز **هیچ رفتار یا فیچر جدیدی اضافه نمی‌کنه**. هدف فقط این بود که همون ربات فاز ۰ —
دقیقاً با همون منطق، همون متن‌ها، همون دکمه‌ها — از یک فایل ۱۵۹۲ خطی به یک ساختار
ماژولار، خوانا و قابل‌توسعه تبدیل بشه (طبق بند ۱ سند نیازمندی‌ها).

## 📁 ساختار جدید

```
main.py                        # نقطه ورود — python main.py
signal_bot/
├── config/settings.py         # همه‌ی تنظیمات (env + مقادیر ثابت بازی)
├── db/
│   ├── connection.py          # اتصال SQLite + ساخت/آپدیت جداول
│   ├── users_repo.py          # همه query های خام جدول users
│   ├── signals_repo.py        # همه query های خام جدول signals
│   └── prize_repo.py          # همه query های خام prize_pool/seasons/winners
├── services/
│   ├── scoring.py             # منطق سطح‌بندی، امتیاز نتیجه، استریک
│   └── payments.py            # ارتباط با API نوپیمنتس
├── keyboards/keyboards.py     # همه‌ی کیبوردهای inline
├── formatters/texts.py        # ساخت متن‌های HTML (پروفایل/لیدربورد/...)
├── handlers/
│   ├── common.py              # start + guard مشترک + back_main/cancel + fallback
│   ├── profile.py             # لیدربورد، آمار، پروفایل، تابلوی افتخار
│   ├── signals.py             # ویزارد ثبت سیگنال (دکمه‌ها + مراحل متنی)
│   ├── payments.py            # استخر جایزه، دونیت، فاکتور، چک پرداخت
│   ├── admin.py                # کل پنل ادمین (بررسی/نتیجه/کاربران/آمار/دوره)
│   └── text_router.py         # تنها MessageHandler — بر اساس step دیسپچ می‌کنه
└── jobs/scheduled.py           # جاب پست روزانه + چک رتبه (۶ساعته)
```

## 🧭 چرا این تفکیک؟

- **`db/`** دیگه هیچ SQL خامی بیرون از این پوشه نیست. قبلاً کوئری‌ها داخل
  `button_handler`/`text_handler` پخش بودن؛ حالا هر دامنه (کاربر/سیگنال/جایزه)
  فقط از طریق تابع‌های `*_repo.py` با دیتابیس حرف می‌زنه.
- **`services/`** منطق تصمیم‌گیری (سطح، استریک، امتیاز) از دیتابیس و از تلگرام جداست —
  یعنی فردا اگه فرمول Alpha Score عوض بشه (بند ۸ سند)، فقط همین‌جا دست می‌خوره.
- **دکمه‌ها (`CallbackQueryHandler`) به‌جای یک `if/elif` غول‌پیکر ۵۰۰ خطی، به ۶ پترن
  مجزا و غیرهم‌پوشان تقسیم شدن** (جهت، ناوبری مشترک، پروفایل، سیگنال، پرداخت، ادمین)
  — هرکدوم مستقیماً توسط تلگرام به تابع دامنه‌ی خودش روت می‌شن. یک fallback دفاعی هم
  آخر ثبت شده که اگه callback_data ناشناخته‌ای (مثلاً از یه فیچر جدید ناقص) بیاد،
  به‌جای سکوت یا کرش، توی لاگ ثبتش می‌کنه.
- **پیام‌های متنی** چون بر اساس `step` ذخیره‌شده در `user_data` مسیر پیدا می‌کنن (نه
  خود متن پیام)، نمی‌شد مثل دکمه‌ها pattern-based جداشون کرد. به‌جاش یک
  `STEP_HANDLERS` دیکشنری در `text_router.py` هرکدوم رو به تابع صاحب دامنه‌ش
  (`handle_step_coin`, `handle_step_broadcast`, ...) delegate می‌کنه.

## ✅ تست‌هایی که قبل از تحویل انجام شد

چون امکان اجرای واقعی در برابر سرورهای تلگرام رو ندارم (بدون دسترسی شبکه به
api.telegram.org)، این‌ها رو به‌صورت خودکار تست کردم:

1. **Syntax:** `py_compile` روی تمام فایل‌ها.
2. **Import کامل:** همه‌ی ماژول‌ها import شدن — بدون خطای circular import یا نام تعریف‌نشده.
3. **ساخت واقعی Application:** `ApplicationBuilder().token(...).build()` + ثبت هر ۹
   هندلر + هر ۲ جاب زمان‌بندی، بدون خطا.
4. **پترن‌های callback:** هر ۵۶ مقدار ممکن `callback_data` (تمام دکمه‌های ربات) چک شد —
   هرکدوم دقیقاً به یک هندلر می‌رسه، نه صفر و نه بیشتر از یکی.
5. **جریان کامل کاربر (end-to-end):** ثبت کاربر → ثبت سیگنال → تأیید → ثبت نتیجه
   (`win_10x`, `win_2x`) → آپدیت امتیاز/سطح → بونوس استریک در سیگنال سوم (دقیقاً طبق
   `STREAK_BONUS`) → همه‌ی فرمترها (`user_stats_text`, `leaderboard_text`,
   `prize_pool_text`, `hall_of_fame_text`, `my_signals_text`) بدون خطا و با اعداد درست.

⚠️ این‌ها منطق و import رو تضمین می‌کنن، ولی جایگزین تست واقعی روی تلگرام نیستن —
قبل از دیپلوی نهایی، حتماً خودت با توکن واقعی و در یک چت تست، همه‌ی منوها رو
دستی کلیک کن.

## 📌 چیزی که عمداً تغییر نکرده (باگ‌های شناخته‌شده از قبل)

- `notify_rank_changes` هنوز فقط داده می‌خونه و هیچ اطلاع‌رسانی‌ای انجام نمی‌ده —
  همون‌طور که در بررسی اولیه گفته شد. چون فاز ۱ فقط ساختار رو عوض می‌کنه نه رفتار رو،
  عمداً دست‌نخورده مونده (کامنت توضیحی توی `jobs/scheduled.py` گذاشته شده).
- منطق پرداخت هنوز به کلیک دستی «چک وضعیت پرداخت» وابسته‌ست (بدون webhook) —
  این هم برای فاز ۵ (سیستم مالی) گذاشته شده.
