# ممیزی نهایی قبل از اولین اجرای واقعی

قبل از اولین بار که واقعاً با یک توکن واقعی اجرا کنی، یه دور دیگه و این‌بار با
تمرکز رو چیزهایی که *فقط* در تعامل واقعی با API تلگرام خودشون رو نشون می‌دن
(نه در سطح منطق دیتابیس) مرور کردم. ۳ دسته مشکل واقعی پیدا و رفع شد.

## 🐛 مشکل ۱ (جدی، محتمل) — HTML escape نشدن متن آزاد کاربر

همه‌جای ربات از `parse_mode=HTML` استفاده می‌کنه. متن آزاد کاربر (اسم کوین در
Fast Call، توضیحات سیگنال، نام کامل تلگرام، دلیل پاداش، متن پیام همگانی) در
~۲۰ جای مختلف مستقیم و بدون escape داخل تگ‌های HTML قرار می‌گرفت. اگه این متن
کاراکترهای `<`, `>`, `&` داشته باشه (که کاملاً معمولیه — مثلاً «قیمت < ۵۰۰۰۰»
یا «A&B» یا حتی یه ایموجی متنی مثل «<3»)، تلگرام کل پیام رو با خطای
«can't parse entities» رد می‌کنه.

**رفع شد:** یک تابع مشترک `signal_bot/utils.py::esc()` ساختم و دور *هر* متن
آزاد کاربر که وارد یه پیام HTML می‌شه گذاشتم — اسم‌ها، یوزرنیم‌ها، اسم کوین،
توضیحات، دلیل پاداش، متن broadcast. **تست شد** با رشته‌های واقعاً خطرناک
(شامل تگ `<script>`، `<b>` تزریق‌شده، و `<`/`&` خام) در پروفایل، لیدربورد،
Fast Call، Full Signal متنی، و پاداش — همه‌جا درست escape شدن.

## 🐛 مشکل ۲ (محتمل، رایج) — دوبار زدن یک دکمه کرش می‌کرد

اگه کاربر روی دکمه‌ای بزنه که محتواش دقیقاً با چیزی که از قبل رو صفحه هست یکی
باشه (مثلاً دوبار پشت‌سرهم زدن همون تب لیدربورد)، تلگرام خطای
«Message is not modified» می‌ده. این رفتار طبیعیِ خیلی محتمل کاربرهاست
(دوبار تپ کردن)، ولی هیچ‌جای کد مدیریت نمی‌شد.

**رفع شد:** یک error handler سراسری (`app.add_error_handler`) اضافه شد که این
خطای مشخص رو بی‌خطر و ساکت نادیده می‌گیره، و بقیه‌ی خطاهای واقعی رو با جزئیات
لاگ می‌کنه (قبلاً هیچ error handler ای نبود). **تست شد** با یه BadRequest واقعی
شبیه‌سازی‌شده — هم حالت بی‌خطر، هم حالت خطای واقعی.

## 🐛 مشکل ۳ (جزئی) — طول کپشن سیگنال‌های در انتظار می‌تونست از حد تلگرام رد بشه

توضیحات سیگنال در نمایش «سیگنال‌های در انتظار» به ادمین، بدون سقف طول اضافه
می‌شد. وقتی سیگنال از نوع Full Signal با عکس باشه، این با `send_photo` می‌ره که
سقف کپشنش ۱۰۲۴ کاراکتره (نه ۴۰۹۶ مثل پیام معمولی) — یه توضیح طولانی می‌تونست
باعث بشه اون پیام اصلاً به ادمین نرسه (بی‌صدا، چون `safe_send_photo` خطا رو
می‌گیره ولی سیگنال دیده نمی‌شد). به ۲۵۰ کاراکتر محدود شد.

## ✅ تست‌های نهایی

Syntax + import کامل (شامل `main.py`)، ساخت Application با ۱۳ هندلر +
۱ error handler، رگرسیون کامل فاز ۱ تا ۶ با mockهای دقیق (`spec=telegram.Message`
برای جلوگیری از قطعیت کاذب mock)، و تست جامع escape با ورودی‌های واقعاً خطرناک.
